Class 07: Electronic Input Devices

Assignment

1. Use a sensor from the kit to measure a physical quantity with your microcontroller. Calibrate your sensor (describe the range between its min and max values, and whether you can control this range). Show data in the form of a table or graph.

2. Do the same as (1) with a sensor you have fabricated yourself using copper tape, aluminum foil, Velostat, etc.

Background Knowledge

I thought the best place to start would be thinking about which electronic input devices I would be most likely to integrate into my final project design, deciding that a proximity sensor soul have some useful applications in an RFID scanner. I know from experience that when designing antenna configurations for NFC tags it is often useful to know at what distance away they have been succesfully scanned, and to be able to compare those to simulated values generated in programs like HFSS.

Deliverables: IR_Proximity Sensor

I first wanted to simulate a potentional proximity sensor in TinkerCAD, my final design can be downloaded here. I wanted to first test it be designing a circuit that could succesfully register the phototransistor as a digital input that could be sensed whether or not it was recieving any light and signal the output LED in the top left to turn on if succesful. After this I switched the phototransitor over to an analog input to be able to calibrate its readings and generate a scale od real distances it was measuring. Obviously this would be a lot more useful when building the real circuit. Finally I used BlocklyDuino to test some of my initial ideas for the code to run the circuit.

Calibration

Here I took a page out of Silvia's book from last year and built covers for the emitting LED and the phototransistor using some thick black Gorilla tape. For some reason (possibly a miswiring) the circuit was still reading consitently low (on the scale of 0-20) until I flipped the phototransistor around. This worked but lead to some issues later on which I will discuss.

After this I was able to get some succesful readings and was able to calibrate it such that the "success" LED would be triggered at a reading of around 50.8mm or 2in (a good standard read range for RFID). One issue is that it appears as though the calibration is backwords (I'm assuming because I flipped the phototransistor in the breadboard) such that a greater distance corresponds with a higher signal value. This appears to be the opposite of what it should be doing. My calibration also appears to be overly linear on a scale of 20-120 as opposed to somewhat exponential on the scale of 0-1024. My hypothesis would be that this has something to do with the value of resistors incorporated into the circuit.

Deliverables: Capacitive Sensor

I wanted to start with a super simple capacitive sircuit to make sure I was on the right track. The idea was to make a capacitive proximity Circuit to be able to compare the sensitivity and ease of calibration to the IR proximity sensor.

I started with this design following the example used in class but adding a second 10mOHM in series to increase the sensitivity.

On my initial testing the single plate did indeed respond to my hand (with an RFID tag) acting as the ground at the aproximate range (2-4in) provided by the arduino capcitiveSensor library documentation.

The sensor was obviously more highly responsive to direct touch though. Which provides a good scale for the min/max value we can expect with this setup.
