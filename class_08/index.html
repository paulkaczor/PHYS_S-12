Class 08: Electronic Output Devices

Assignment

1. Use an output device that you haven't used before.

2. Write a microcontroller program that integrates at least one input device and one output device. Avoid the delay() function by using either timers or interrupts.

Background Knowledge and Preview

Similar to the last assignment I figured a good jumping off point would be trying to generate a circuit thatwould be deployable within my final project. I still had some ugrades I wanted to do to my IR proximity sensor to integrate it into an RFID reader and wanted to think about which of the output devices we had practiced with would pair well with that. I started by trying to get the TFT display to work during the workshop period of class as well as a lab but was only able to get it to power on and display a dimmed white screen despite being fairly confident it was wired correctly. After this I moved on to integrating the buzzer as an additional indicator of succesful tag reads alongside the check LED used in my previous assignment.

Deliverables: Output Device

I started off by making some upgrades to the proximity sensor input device I still had wired up. This included tweaking the resistance (up to a 10kOhm) leading up to the phototransistor to try and acheive a better scale for the total readable distance and recalibrating it to find better values for read distances.

This made the proximity sensor much more sensitive and I was able to get the check LED to signify an appropriate distance more easily.

At this point I wanted to first see if I could get a buzzer integrated into the circuit and able to play a generic myTone() function. I started like always in TinkerCAD. My design is available to tinker here.

I then wired it up for real and this is when the real fun began. I stared noticing a consistent issue with both my MetroExpress as well as the huzzuh boards where once plugged into my computer they would power on but not be recognized as available boards by Arduino IDE. The current hypothesis is that somehow either the boards or the USB ports on my computer are burning out the data transfer capablities of the cables attaching the microcontrollers. As of writing this I have been unable to fix this but luckily I was able to get my buzzer wired up for about three test runs before this started occuring.
